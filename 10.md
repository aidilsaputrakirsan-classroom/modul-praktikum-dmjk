### a. ğŸ–§ Implementasi Topologi ISP Ganda

#### 1. Topologi Jaringan yang Diperkuat
```
          [Router2/ISP1]      [Router3/ISP2]
         Gi0/0:10.10.10.1    Gi0/0:20.20.20.1
                |                   |
        -------Gi0/0-----     ----Gi0/1-----
       |            |         |             |
       |        [Router1/Gateway]           |
       |                 |                  |
       ------------Gi0/2:192.168.10.1--------
                         |
                   [PC0:192.168.10.2]

```

#### 2. Alokasi IP Address
```
| Perangkat | Interface        | IP Address   | Subnet Mask   |
| --------- | ---------------- | ------------ | ------------- |
| Router1   | Gi0/0            | 10.10.10.2   | 255.255.255.0 |
| Router1   | Gi0/1            | 20.20.20.2   | 255.255.255.0 |
| Router1   | Gi0/2            | 192.168.10.1 | 255.255.255.0 |
| Router2   | Gi0/0            | 10.10.10.1   | 255.255.255.0 |
| Router3   | Gi0/0            | 20.20.20.1   | 255.255.255.0 |
| PC0       | Fa0/0 atau Gi0/0 | 192.168.10.2 | 255.255.255.0 |
```

### b. ğŸ”§ Langkah-Langkah Konfigurasi

#### 1. Konfigurasi Router ISP1
```cisco
Router>enable
Router#configure terminal
Router(config)#interface gigabitEthernet0/0
Router(config-if)#ip address 10.10.10.1 255.255.255.0
Router(config-if)#no shutdown
Router(config-if)#exit
```

#### 2. Konfigurasi Router ISP2
```cisco
Router>enable
Router#configure terminal
Router(config)#interface gigabitEthernet0/0
Router(config-if)#ip address 20.20.20.1 255.255.255.0
Router(config-if)#no shutdown
Router(config-if)#exit
```

#### 3. Konfigurasi Router Gateway
```cisco
Router>enable
Router#configure terminal
Router(config)#hostname Gateway
Gateway(config)#interface gigabitEthernet0/0
Gateway(config-if)#description ISP1
Gateway(config-if)#ip address 10.10.10.2 255.255.255.0
Gateway(config-if)#no shutdown
Gateway(config-if)#exit

Gateway(config)#interface gigabitEthernet0/1
Gateway(config-if)#description ISP2
Gateway(config-if)#ip address 20.20.20.2 255.255.255.0
Gateway(config-if)#no shutdown
Gateway(config-if)#exit

Gateway(config)#interface gigabitEthernet0/2
Gateway(config-if)#description Ke LAN/PC
Gateway(config-if)#ip address 192.168.10.1 255.255.255.0
Gateway(config-if)#no shutdown
Gateway(config-if)#exit
```

### c. ğŸŒ Konfigurasi NAT dan Routing pada Router Gateway

#### 1. Konfigurasi NAT dan Routing
```cisco
Gateway(config)#access-list 1 permit 192.168.10.0 0.0.0.255
Gateway(config)#interface gigabitEthernet0/0
Gateway(config-if)#ip nat outside
Gateway(config-if)#interface gigabitEthernet0/1
Gateway(config-if)#ip nat outside
Gateway(config-if)#interface gigabitEthernet0/2
Gateway(config-if)#ip nat inside
Gateway(config-if)#ip nat inside source list 1 interface gigabitEthernet0/0 overload
Gateway(config)#ip route 0.0.0.0 0.0.0.0 10.10.10.1 1
Gateway(config)#ip route 0.0.0.0 0.0.0.0 20.20.20.1 2
Gateway(config)#interface gigabitEthernet0/0
Gateway(config-if)#shutdown
Gateway(config-if)#exit
```

#### 2. Setting PC
- IP: 192.168.10.2
- Subnet mask: 255.255.255.0
- Gateway: 192.168.10.1

## ğŸ› ï¸ Tips Troubleshooting

- **Masalah Konektivitas ISP**:
  - Verifikasi konfigurasi router ISP dan pengalamatan IP
  - Periksa koneksi kabel dan status link
  - Uji dengan ping sederhana ke gateway ISP

- **Failover Tidak Berfungsi**:
  - Verifikasi konfigurasi administrative distance
  - Periksa waktu konvergensi protokol routing
  - Uji penghapusan/penambahan route manual

- **Layanan Publik Tidak Dapat Diakses**:
  - Verifikasi translasi NAT statis
  - Periksa firewall/ACL yang memblokir traffic masuk
  - Uji konfigurasi port forwarding

- **Masalah Kinerja Cloud**:
  - Monitor utilisasi bandwidth
  - Periksa packet loss atau latensi tinggi
  - Verifikasi kebijakan QoS untuk traffic cloud
